THE 3 STORAGE LAYERS
Embedding Model
all-MiniLM-L6-v2 (sentence-transformers) · runs locally on CPU · no API key
Input: any text string  →  Output: 384 float numbers (a vector) · ~5ms per call
Used in: cache check (step 3) AND schema retrieval (step 4)
ChromaDB (chroma_db/)
Collection: schema_metadata
15 rows — one per table. Stores table description + columns as vectors. Used in step 4 to find relevant tables.
Collection: query_cache
Stores past question→SQL pairs as vectors. Used in step 3 for instant cache hits. Similarity threshold = 0.92.
Does NOT contain actual business data. Only descriptions + past queries.
SQLite (enterprise.db)
Finance: invoices (200), payments (150), expenses (80)
Sales: customers (100), sales_orders (120), contracts (30)
Procurement: vendors (40), purchase_orders (100)
HR: employees (50), departments (7), leaves (60)
Inventory: products (60), inventory_transactions (80)
INTERVIEW CHEATSHEET — ONE-LINERS
ChromaDB
Vector database. Stores table descriptions + past queries as 384-number arrays. Finds similar text using cosine similarity.
Mistral 7B
Local LLM via Ollama. Called twice per query: once to generate SQL, once to explain results in plain English.
NetworkX
Graph library. FK relationships = edges. Shortest path between tables = auto-discovered JOIN conditions. Zero hardcoding.
Ollama
Runs Mistral 7B locally at localhost:11434. No API key, no internet, no cost. We POST prompt, get text back.
SQLAlchemy
ORM that executes generated SQL on enterprise.db. Returns list of dicts. Has safety layer (SELECT only).
Semantic Cache
If new question is 92% similar to a past question → return cached SQL instantly. Skips ALL LLM calls. ~50ms vs ~3500ms.
Intent Router
Mistral classifies every input as DATA or CHAT. DATA → SQL pipeline. CHAT → direct conversational reply.
MiniLM-L6-v2
Embedding model. Converts any text to 384 numbers locally. Used by ChromaDB automatically for all vector operations.
Cosine Sim
Measures angle between two vectors. 1.0 = identical meaning, 0.0 = opposite. We use 0.92 as cache threshold.
SSE Streaming
Server-Sent Events at /query/stream. Tokens stream to browser in real time like ChatGPT. Uses FastAPI StreamingResponse.
Session ID
Multi-turn conversation. Each session stores last 10 turns in memory. Follow-up questions use previous context.
Confidence
Decreases per retry: 1.0 → 0.85 → 0.70. Shows how many attempts Mistral needed to produce valid SQL.
